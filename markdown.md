# github中markdown文本输入模式的语法及使用
## 1.Markdown文本编辑模式介绍
Markdown是一种轻量级标记语言，创始人为约翰·格鲁伯（英语：John Gruber）。它允许人们“使用易读易写的纯文本格式编写文档，然后转换成有效的XHTML（或者HTML）文档”。  
自从GitHub流行以来，Markdown作为一种轻量级标记语言就深受程序员的喜爱，上手也十分简单。  
## 2.markdown语法
### 2.1多级标题
Markdown支持多级标题对应不同的字体大小，标题级别越小其代码前的#越多，最多六个，如下：  

代码块：
```
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
```
显示如下：
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
---
需要注意的是，这里的#开头的文字或标题必须另起一行，#和文字中间必须有空格，不然GitHub的README无法识别。
#### 除了用#标识字体(标题)大小，还有一种方法可以标识一级和二级标题：

代码块：
```
一级标题
=========
二级标题
---------
```
显示如下：

一级标题
=========
二级标题
---------
***
这里的字体只分为两级，大号的字体下一行用"="号隔开，略小一号的字体用"-"（短横线）隔开，"="或"-"的数量最好是三个以上，但是有些版本可以为一个，通常的习惯为与字体等长。
### 2.2段落和换行
#### 分隔线
在实际的文本显示中，为了使文本更加清晰明了，可以适当地加入一些分隔线如下：

代码块：
```
***
---
```
显示如下：

***
---
分隔线由***或者---表示，一般是三个连续的符号，单行开头，也可以多于三个
#### 换行
换行可以使用一个或者多个空行来另起一个段落（注意是一个空行，而不是简单地用回车来另换一行），或者在上一行的行尾添加两个空格，然后以回车来换行。如下  

代码块：
```
第一行

第二行
第三行  
```
或者
```
第一行  
第二行  
第三行  
```
显示如下：

第一行

第二行
第三行  

---
第一行  
第二行  
第三行  
### 2.3插入文本引用
需要将一段文本与正文文本相区分，比如示例，引用文章之类的，需要先换行(隔一个或者多个空行)，然后新行以tab键开头，键入文本。  
代码块：
```
正文

    引用文本示例
```
显示如下：

正文

    引用文本示例
### 2.4列表
#### 无序列表
无序列表使用* + -来标识，注意在字符之后要添加一个空格，示例：

代码块：
```
* Markdown1
+ Markdown2
- Markdown3
```
显示内容：  
* Markdown1
+ Markdown2
- Markdown3

这三种符号是可以交叉使用的
#### 有序列表
有序列表需要添加一个相应的'数字'+'.'+'空格'来标识，示例：

代码块：
```
1. Markdown1   
2. Markdown2  
3. Markdown3  
```
显示内容：
1. Markdown1  
2. Markdown2  
3. Markdown3  
### 2.5代码
单行代码用两个`包含起来即可（这个符号不是单引号，而是反引号，键盘左上角那个）。  
多行代码在代码上下两行分别加```（三个反引号）即可。  
代码块：
``print "hello world"``  
和
```
```
print "hello world" 
print "hello world" 
print "hello world" 
```
```
显示内容：  
`print "hello world" `
和
```
print "hello world" 
print "hello world" 
print "hello world" 
```
### 2.6链接
添加链接的方法为：[名称](网址)，如：  
代码：
`[博客](https://www.cnblogs.com/downey-blog/)`
显示内容：  
[博客](https://www.cnblogs.com/downey-blog/)  
点击高亮字体即可进入相应链接。
### 2.7插入图片
插入图片的方法与插入链接相比仅仅在前面多了一个 "!"。  
代码块：  
`![pic](https://raw.githubusercontent.com/linux-downey/bloc_test/master/picture/Makedown/Makedown.png)`
显示内容：  
![pic](https://raw.githubusercontent.com/linux-downey/bloc_test/master/picture/Makedown/Makedown.png)  
值得注意的是，目前还并没有一种兼容性比较好的方法将图片内嵌到文本中，一般的做法是在文本内引用图片的链接，但是在这种做法下，如果图片资源一旦迁移，将会额外增加修改成本
